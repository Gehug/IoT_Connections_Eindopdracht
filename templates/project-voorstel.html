1     Ons project

 

1.1     Wat willen we precies doen?

Voor het project dat wij gaan moeten uitvoeren voor het vak, IoT Connections hebben wij besloten (Gil en Lowie), om met ons wel bekende Lora te gaan werken. Hier gaan we gebruik maken van het Helium netwerk, aangezien hier een hele hoop gateways over heel het land ter beschikking zijn (zelfs in heel Europa!).

 

Wat zijn nu onze plannen met Lora? Wel dat is erg simpel, wat wij willen doen is eigenlijk een klein platform bouwen waar je tempratuur op een bepaalde locatie kan bekijken. Met andere woorden gaan wij tempratuur en gps-cordinaten doorsturen naar een zelfgebouwde server die een zelfgebouwde REST API bevat (dat wij gaan ontwikkelen met de python module Flask). We gaan dan proberen de sensor data te displayen op een web interface, ook zouden we graag een map functionaliteit toevoegen.

 

1.1.1     Flask API

Zoals we eerder al vermeld hebben gaan we dus d.m.v Lora tempratuur dat versturen, naar een helium gateway. Vanaf dat we de helium-gateway hebben bereikt zullen deze data moeten forwarden naar een API.

 

Welke data hebben we nodig?

  GPS Cordinaten

  De gemeten Tempratuur

  Device EUI

  Tijd gaan we op de API voorzien, omdat esp-32 geen tijd onthoudt

 

1.1.2     Website

Voor de web interface gaan we heel simpel een lijst maken die de laatste tempratuur informatie gaat tonen op de website, samen met de Device EUI en de gps-informatie + tijd.

 

1.1.2.1 Website mockup

Onze website gaat zoiets als dit er komen uit te zien, een lijst van de laatste nieuwe tempratuur waardes. Ook gaan we proberen wanneer je op de sensor data zal klikken dat deze een map zal openen waar de sensor data is opgenomen.

 

1.1.3     Map

 

Voor de map gaan wij proberen om in een straal van een paar kilometer een cirkel op de map te zetten. Afhankelijk van de tempratuur zal de kleur van de bol een kleur krijgen. Bv: Blauw = tussen -5 tot 9 graden | Oranje = 10 tot 24 graden | Rood = 25+ graden.
